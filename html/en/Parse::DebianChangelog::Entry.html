NAME
       Parse::DebianChangelog::Entry - represents one entry in a Debian
       changelog

SYNOPSIS
DESCRIPTION
   Methods
       init

       Creates a new object, no options.

       new

       Alias for init.

       is_empty

       Checks if the object is actually initialized with data. Due to
       limitations in Parse::DebianChangelog this currently simply checks if one
       of the fields Source, Version, Maintainer, Date, or Changes is
       initalized.

   Accessors
       The following fields are available via accessor functions (all fields are
       string values unless otherwise noted):

       ·   Source

       ·   Version

       ·   Distribution

       ·   Urgency

       ·   ExtraFields (all fields except for urgency as hash)

       ·   Header (the whole header in verbatim form)

       ·   Changes (the actual content of the bug report, in verbatim form)

       ·   Trailer (the whole trailer in verbatim form)

       ·   Closes (Array of bug numbers)

       ·   Maintainer (name and email address)

       ·   Date

       ·   Timestamp (Date expressed in seconds since the epoche)

       ·   ERROR (last parse error related to this entry in the format described
           at Parse::DebianChangelog::get_parse_errors.
